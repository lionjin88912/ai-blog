# 台灣繁體中文翻譯風格指南 (Style Guide)

## 語氣與人稱
- **專業正式**：技術文件或商務書信建議使用「您」，語氣客觀中立。
- **親切自然**：UI 介面、部落格文章或一般對話可使用「你」，語氣活潑、具備引導性。
- **避免冗贅**：翻譯時應刪除不必要的「被動語態」或「的一個」等冗長結構。

## 標點符號規範
- **全形標點**：中文句子一律使用全形標點符號（如：，。？！：；）。
- **括號**：中文內容使用全形括號（），若括號內全為英文或數字則視情況使用半形。
- **間隔**：中文與英文單字/數字之間建議加入一個半形空格，以提升閱讀舒適度（例如：「執行 3 次」而非「執行3次」）。

## 程式碼與技術處理
- **保留原文**：程式碼、變數名稱、路徑、指令（如 `git commit`, `npm install`）請務必保留原文，不要翻譯。
- **註解處理**：翻譯程式碼註解時，請確保不破壞代碼結構。
- **標題與選單**：UI 選單項目翻譯應簡潔有力，避免過長。

## 範例
- **Bad**: 點擊這個按鈕去優化你的項目的質量。
- **Good**: 點選此按鈕以最佳化您的專案品質。
